# materials/ — 原始参考资料(不发布)

放这里的东西**不会**进入文档站点(mkdocs 只打包 `docs/` 目录)。用于存放:

- Pico4 Ultra 企业版 APP 使用说明原件(PDF / docx / 原始截图)
- 数采启动流程说明原件
- 参考手册、协议文档、原始录屏等

## 工作流

1. 把手上的资料**原样**丢进本目录(可自建子文件夹,如 `materials/pico4/`)。
2. 告诉我放了什么;我从中挑图、裁剪/压缩后另存到 `docs/assets/<主题>/`
   (发布用),并写进对应章节(§3.4 Pico4、§3.6 上电流程等)。
3. 原件留在这里备查,不进站点。

> 约定:**原始/大体积/含内部信息**的资料留在 `materials/`;**最终发布的图片**
> 放 `docs/assets/`。二者分开,避免把内部原件误发布。

## 版本库

`materials/` 默认被 `.gitignore` 排除(原件常较大或含内部资料),仅保留本 `README.md`。
如需版本化某些素材,可在 `.gitignore` 里加白名单(`!materials/xxx`)。
