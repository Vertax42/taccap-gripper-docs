# 采集规范与最佳实践

会跑 `lerobot-record` 只是第一步;**采到"好数据"**才决定下游价值。本章讲怎么让落盘的
`LeRobotDataset` 干净、一致、可用。建议采集前通读一遍,采集中对照自查。

!!! tip "一句话原则"
    每条 episode 都应是一次**完整、平稳、传感器有真实信号**的演示,且整份数据集处于
    **同一坐标原点**下。

## 什么是一条"好 episode"

| 维度 | 好 ✅ | 差 ❌ |
|---|---|---|
| 完整性 | 从接近 → 抓取 → 操作 → 完成,任务闭合 | 中途手抖中断、没完成 |
| 平稳性 | 手持带动平稳、匀速 | 急停急转,IMU/位姿剧烈抖动 |
| 触觉信号 | 抓取时传感器与物体**真实接触**、有形变 | 空抓、没接触、触觉图无信号 |
| 相机可见 | 目标在腕相机视野内、不被遮挡 | 手/线缆遮挡目标,或过曝/频闪 |
| 坐标一致 | 与本数据集其它集**同一原点** | 中途重启 XenseVR-Toolkit,原点漂移 |
| 夹爪读数 | `gripper.pos` 闭合≈0、张开合理 | 未标定,闭合不为 0 |

## 采集前

- **对齐并冻结坐标系**:**戴头显启动 XenseVR-Toolkit 时面朝机器人正前方**(冻结世界系方向:X 前 / Y 左 / Z 上)。
  启动瞬间原点与方向被冻结,**整份数据集采集期间绝不重启客户端**——一旦重启参考系就变了;若不得不重启,
  把之后的数据当作**新数据集**处理。
- **确认标定有效**:编码器零点已锁定(闭合读数≈0);见 [标定与自检](04-calibration.md)。
- **自检设备**:`scan_grippers` 输出 side/role/firmware_sn 正常;追踪器有电、有位姿。
- **关闭 WiFi**:Pico4 走有线共享网络,数采时**关闭数采电脑 WiFi** 避免网络冲突(见 [3.4 网络连接](03-host-hardware.md#pico-network))。
- **准备场景**:光照稳定、目标清晰可见;清理会遮挡腕相机的线缆/杂物。
- **预留磁盘**:满负荷出流可达 ~280 MB/s(双臂),确保磁盘空间与写入带宽充足。

!!! note "养成习惯:开 Rerun 自查"
    采集时加 `--display_data=true`,实时看夹爪轨迹、`gripper.pos` 与触觉图是否有信号,
    第一时间发现空抓、漂移、丢帧。

## 演示动作规范

- **完整**:每条演示覆盖任务全过程,自然收尾,不要半途而废。
- **平稳匀速**:手持带动要稳,避免急停、急转、甩动——这些会在 IMU 与位姿上留下噪声。
- **真实接触**:抓取阶段确保视触觉传感器与物体接触、产生形变信号(触觉是本设备的核心模态,
  空抓的演示价值很低)。
- **节奏一致**:同一任务的多条演示,动作节奏尽量一致,便于模型学习。

!!! warning "shifted-frame:别把关键动作压在第 0 帧"
    录制采用移位帧配对,**每集会丢掉第一帧**(首帧无前驱)。所以开录后先稳定 0.5~1 秒
    再进入关键动作,并给足 `--dataset.episode_time_s`,避免关键信息落在被丢的首帧。

## 分集与复位

- 集与集之间是**被动复位**:重新摆放物体/场景,无需遥操。
- 每条 episode 是**一次完整演示**;不要把多次尝试塞进一集。
- `--dataset.episode_time_s` 给够但别过长——过长会产生大量无效尾帧,拖慢与稀释数据集。

## 多样性 vs 一致性

好数据集在**任务语义上一致、在无关因素上适度多样**:

- **保持一致**:任务定义(`--dataset.single_task` 描述)、动作意图、坐标原点。
- **适度多样**:物体初始位姿/位置、抓取点、光照的轻微变化——有助于下游泛化。
- 不要在同一份数据集里混入**不同任务**;不同任务/明显不同的变体应拆成**不同数据集**。

## 任务定义与数据集组织

- **`single_task` 用稳定、清晰的英文短句**(如 `'Pick up the object'`),同一数据集内保持一致——
  它会写进数据集的 `tasks`。
- **一个数据集对应一个任务/变体**;`repo_id` 命名建议 `<org>/<任务>_<日期>`,便于检索与管理。
- 记录本次采集的**设备台账**(用哪只夹爪/追踪器、标定时间),便于回溯。

## 增量采集:先小批、验证、再规模化

不要一口气采几百条再回头发现系统性问题。推荐节奏:

1. 先采 **5~10 条**。
2. `lerobot-check-dataset --repo-id <...>` 校验(见 [数据集与示例](06-dataset.md#62))。
3. 回放/可视化抽查几条,确认动作、触觉、位姿都正常。
4. 确认无系统性问题后再**规模化**采集。

## 常见坏样本与规避

| 坏样本 | 症状 | 规避 |
|---|---|---|
| 坐标漂移 | 位姿在集间跳变 | 采集全程不重启 XenseVR-Toolkit |
| 空抓 | 触觉图无信号 | 抓取时确保真实接触、看 Rerun 触觉 |
| 遮挡 | 腕相机看不到目标 | 清理遮挡、调整手持角度 |
| 抖动 | IMU/位姿噪声大 | 平稳匀速带动 |
| 丢帧 | 日志丢帧告警 | 增大 `encoder_threads`、`vcodec=auto`([5.5](05-data-collection.md#55)) |
| 未标定 | 闭合 `gripper.pos`≠0 | 采集前重标零点([4.1](04-calibration.md#41)) |
| 首帧丢关键动作 | 动作起点异常 | 开录后稳定 0.5~1s 再动作 |

出现报错先查 [故障排查](troubleshooting.md)。

## 采集自查清单

- [ ] XenseVR-Toolkit 本次采集期间**从未重启**(原点一致)
- [ ] 编码器零点有效(闭合 `gripper.pos`≈0)
- [ ] 追踪器有位姿、Rerun 轨迹正常
- [ ] 已关闭数采电脑 WiFi(只留 Pico4 有线共享网络)
- [ ] 抓取时触觉图有信号(非空抓)
- [ ] 腕相机视野内目标不被遮挡
- [ ] 无丢帧告警
- [ ] `single_task` 描述与本数据集一致
- [ ] 磁盘空间充足
